2 Introduction

2.1 Purpose of this Document

The purpose of this document is to provide the reader with information on the test objectives, methodology, test coverage, schedules, criteria and requirements of the GB18030 conformance test for IBM Rational Developer for System z 9.5 on Chinese environment as described below.  This test will be performed by China Development Lab (CDL).

2.2 Scope

The test plan will list all assertions required for testing the IBM Rational Developer for System z 9.5.

The objective is to verify the following:

・         Existing functions work correctly for GB18030 conformance in the localized environment.

The test plan will also detail the environment requirements to run the tests.

6 Defect Management

6.1 Defect Identification

Problems with IBM IBM Rational Developer for System z 9.5 will be identified based upon analysis of test case results and other observations of the tester. Product operation which contradicts the product documentation and GB18030 compliance will be considered as a defect.

The originator of a defect must provide complete and detailed descriptions of any problem found, as well as instructions on problem reproduction, the build level used when the failure occurred, the type of hardware and software being used, etc.

6.2 Defect Reporting and Tracking

The problems found during the IBM Rational Developer for System z 9.5 GB18030 conformance test will be reported as defects using RTC System. The RTC Defect System will be used to report suspected problems, and to track those problems and their resolutions.

All defects must be prefixed by "GB18030" in the abstract. If a defect is determined to be blocking defect, the originator should use the keyword tab, entering both Blocking and GB18030 as keywords for more proficient query.

Appendix B: Defect Definition for GB18030 Compliance Test

Severity

Description

Supplementary description for GB18030

1 - Critical

Catastrophic or complete system failure; any fatal error where the application or operating system hangs, unrecoverable application error, a function that is not available (cannot open) for testing.

Critical I18N design issue that is required any design change to resolve, that leads to the product is not aware of GB18030.

2 - Serious

The system or application fails to correctly process a transaction causing an incorrect transaction to occur; loss or corruption of data; incorrect output; breach in security; unacceptable application response times.

loss or corruption of GB18030 data; GB18030 character rendering problem on all of product GUIs.

3 - Moderate

Output are misleading or redundant; partial failure (mouse, keyboard, short-cut keys, etc.), poor response times.

GB18030 character rendering problem, such as all GB18030 chars displayed as square marks.

4 -Mild

Misspellings; incorrect grammar; poor readability due to font size or type; suggestions for improvement.

Minor display problem.
